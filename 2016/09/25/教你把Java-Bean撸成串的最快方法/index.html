教你把Java Bean撸成串的最快方法

发表于 2016-09-25   |   分类于 Java   |     |   阅读次数

适用工具：IntelliJ IDEA，Android Studio

1 效果展示

废话少说，看效果。

我相信，很多人都已经熟悉链式调用了。如果不熟悉也没有关系，链式调用就是我上面的写的那个样子。你在写的过程中，可以一直点下去。是不是很爽!

但是，通常情况下，Java中只有建造者模式才提供这种方法，另外还有些框架和库提供。

想想使用RxJava的人，写代码真是爽的不得了。当然RxJava那个爽的另有其因，它把异步撸成了串的同时把多层嵌套撸成了串。要知道，我的Leader说(甩锅用的，后面说的不对不要找我)，人思维的嵌套层次大概只有3层，如果再多，代码阅读会非常困难。而代码嵌套在异步处理的时候经常发生。所以RxJava充分解决了这个问题。好像扯远了。

啪!说好的废话少说呢？

2 原理

在我看来，我们在写Java Bean的时候，经常写的set方法，其中的返回值void，简直就是一种资源浪费。所以我想，将set方法返回this，不就可以实现Java Bean的链式调用吗？嗯，没错，我上面的链式调用就是更改了Java Bean中的set方法，让他们返回this。那么看一下Student的代码。

对就是这个样子。

但是我们在写代码的时候通常都是自动生成setter和getter方法的呀。我之前都是生成完了setter方法之后，全局替换void字段，但是这种效率太低了，而且，当你的Bean中有其他方法了之后，就不能这么干了。那么接来下展示 神技 !

3 神技效果

看图吧!

生成的setter方法正是我们需要的。前面已经写了，该神技适用于IntelliJ IDEA，Android Studio。其实就是IntelliJ IDEA内核嘛。

那么来看看怎么做吧!

4 神技实现过程(自定义setter脚本)

是不是感觉高大上了，其实很简单。这就教给大家。

4.1 打开Generate并进入setter

4.2 进入Setter成员选择界面

这个界面你是不是再熟悉不过了，但你有没有注意看过我标记的两个地方？

Template里就是setter的模版生成脚本，我们当前使用的是默认的。这个时候如果你回去上面的动图，会发现，我的这里是my setter。其实就是我自己写的一个生成脚本。但是后来我发现，根本没有这种必要因为，IntelliJ IDEA已经给我们准备好了。

点击下拉键，选择Builder模版(这个模版就是带有this返回值的setter生成模版)，然后再选择成员，确定，即可生成我们要的setter。

如果有必要的话，我们还可以自己自定义模版。

4.3 自定义模版

点击…，进入模版管理界面。

我们可以通过＋创建一个新的模版脚本，然后编写脚本即可。之后点击Ok。我自己写了一个my setter，后来发现和系统的Builder一样。囧!

5 结束语

其实不止是setter的模版可以这样改，其他的也可以，大同小异。去按自己的需求做吧!
