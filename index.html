离开学校的我却更会学习了

发表于 2016-09-10   |   分类于 ∆   |     |   阅读次数

阅读本文你可能需要1.5分钟，也可能需要15分钟。

离开学生生涯大概2个月了，我发现自己更会学习了。

其实想想，在自己还没有什么成功事迹的时候，讲这种东西也没有什么客观说服力。但我已经觉得自己越来越好了。那么下面分享一下我是怎么做的。

分析自身特点

没有一种方法是万能的。所以第一件事情就是对自身的特点进行分析，然后量身打造。那么我的特点是：

  1. 我长的不帅
  2. 我是一个程序员
  3. 我没有女朋友

那么好，深入分析一下我的特点。

第一点，虽然我不承认我长的丑，但我承认我确实长的不帅。这就导致我不能靠脸吃饭。当然，如果可以谁不愿意呢？

第二点，我是一个程序员。对，就是你们常说的程序猿，只不过我穿鞋，梳头，幽默。做为一个不帅的程序猿我只能靠实力生存下来，这也要求我需要不断的去学习。

第三点，我没有女朋友，补充一点，也不是gay。这要求我更要爬上去，不然在这个物欲横飞的社会中，我连交配权都没有。

总结分析结果，我必须要有实力，学习是我的方式。而且如果在读的你够帅或者已经有女朋友了，就不用往下读并参与和我的竞争了，狗急了可是会咬人的。

学习

我的自身特点决定了我必须要通过学习来增加我的实力。那么就要讲讲我是怎样学习的。

最平常的状态就是我最差的状态

根据我对自己生活的反思，我发现最平常的状态却是我最差的状态。

为什么这么说呢？因为我发现，无论我是突然发现自己太菜而忧虑重重也好，还是做的事情得到认可受到激励也罢。我都会在接下来的一段时间里更加努力，学习效率也非常高。唯独在自己平常的时候，漫无目的，都不知道自己整天在做些什么，又不以为然。现在想想真的可怕，所以我说，最平常的状态就是我最差的状态。

而且我也清楚的知道，我平常的状态，学习效率并没有多高。我还不是那种已经把高效养成习惯的人。这也正是为什么我会在忧虑重重或受到正反馈之后，只能将高效率学习维持一段时间的原因。那么既然没有这种习惯，我选择让自己持续性的收到打击或者正反馈。总之就是不要处于平常的状态。那么多久一次呢？要怎么做呢？

周总结

根据自身的特点，我在受到打击或正反馈之后的那一股劲，一般不到一个星期。嗯，就是这么菜。那么我要做的就是周总结。一周基本上是我可以维持一个状态的一个时间单元。而总结正是让自己，受到打击或者正反馈的方式。通过总结这一周自己学习到的东西，收获的东西，失去的东西来给自己这种信息。

周总结的内容

这个其实并不是一个死模版，而是不断变化的。我的也是在不断的更改，把没用的去掉，把有用的加上。那么我讲一下我现在周总结的内容：

题目

[舒适区or学习区or恐慌区]+第几周

如果对舒适区，学习区，恐慌区还不了解的人，可以随便上网搜索一下它们的意思。回想一下，其实我上面讲的最平常的状态，都是处于舒适区。

那么为什么要这两个内容呢？

中括号中的内容醒目，可以一目了然自己这一周处在怎样的状态。这让自己在之后回顾的时候有一个快速的认识。而不需要每次把这篇周总结读一遍，再分析出自己这周所处的状态。

后面的第几周，本身就是对自己的一种正反馈。因为坚持难能可贵。当你发现后面的数字在不断的变大，你也会为自己的坚持感到自豪。

工作相关

  1. 做了什么事情
  2. 遇到什么问题，如何解决的

第一点是对自己工作的一种总结，当然不限于工作，由于工作在我的一天当中占的比例最大，所以这里大多是我工作的事情。这个可以对自己工作有一个宏观的认识，而且，如果你这一周承担了一项对于公司很重要的工作，并完成了，这本身也是一种正反馈。

第二点是对问题的总结，这里不仅限于工作内容的具体问题，比如学习方法，以及什么情况发生让你效率很低，这也是问题。而这里总结一周遇到的问题，以后改正，让自己不再犯同样的错误。

学习总结

新学习到什么东西？

注意是新，老的不算。这一般是我受到打击和激励的直接内容，如果我发现这一周，在这一块几乎写不出东西，那么我心里一定不会很舒服。

最终总结

主要是为下一周做决策：

  1. 哪些是要继续保持的
  2. 哪些是应该舍弃的
  3. 哪些是要避免再此发生的。

目标

王健林给年轻人的建议嘛，我先挣它一个亿。目标对于我们很重要不然就像一个没头苍蝇。而且当你完成了自己的目标你也会收到正反馈。一般我的目标期限是一周。

暂时我的周总结是这些内容，回顾一周的生活，完成这份周总结所用时间大约为40分钟到1个小时。每周日晚上10点进行。

学习缓冲区

周总结每周都在进行，每周都有自己的目标任务。而对于我本身的情况，业余时间不多，如果每遇到一个额外的问题就进行深入研究，可能会消耗大量的时间。(以前我就是这么做的)该大量时间其实也不叫大量，可能是一天的业余时间，也可能是两天。于是乎，自己既定目标没有办法完成。

比如A事情是你定下一周要完成的任务，有的时候你干了一天A事情，突然B事情来了，你在B事情上花了两天时间，这个时候再回来干A事情，你的激情和状态可能已经不同了，而且你还需要回顾你上次A事情做到哪里了。然后你又做了A事情一天，这个时候C事情又来了，你在C事情上又花了一天，那么你的A事情被不断的延期，直到你不想再做A事情。到最后给自己找个借口说，现在没有时间，等以后找个大块的事情再来搞A吧。然后你发想你根本就不会有大块的时间，因为你的时间都被那些零碎事情碎片化了。

所以我为我的学习创建了一个学习缓冲区。当我工作中遇到了诸如B事情或者C事情的时候，我总是适可而止。比如一个bug出现了，我在stackOverFlow上找到了解决办法，解决了问题，OK了。如果这个问题我觉得有深入研究的必要我也不会马上去进行，而是把它记录到自己的学习缓冲区中。然后工作可以继续正常进行，业余时间继续做自己本周的任务。然后每周结束的时候看一下学习缓冲区中的任务，如果大于等于4个，下一周目标任务就是清空学习缓冲区。

One By One

再有一点，无论是工作还是业余时间学习，都要遵循一个原则，就是事情一件一件来。有很多人觉得眼观六路耳听八方，同时可以做几件事情非常了不起。嗯，我承认，确实了不起。但我的确是一个一般人，也不认为自己是个超人。有一段时间我深受一件事情被打断，然后做另一件事情，之后再回来继续做事情的困扰。而且有的时候，不是外在原因，而是因为自己。

那么最好的办法就是，每天早上上厕所的时间，思考一下今天要做的任务，然后列一个清单。如果有必要把时间也写上。我在这里使用的滴答清单这个软件。而且这种软件还有很多，可以随便找，用哪个无所谓。之后就是一件一件的完成今天的所有任务。这样没有打断的工作或者学习，效率杠杠的。当你遇到有要打断你工作的事情，如果是自己的事情，你可以先放一放，在里面创建一个新的任务，之后解决。如果涉及其它人紧急的事情，那么你可以去做，回来的时候看清单中你正在做的任务，你也可以比较快的回忆起来。

收藏

对于我自身而言，我在互联网上的时间可以说占到了我生活的85%以上。因为我每周还会出去玩一玩，哈哈。正常工作的话，基本整天都会在互联网上。那么我在网上浏览的内容很多，我的大多数信息都是来源于互联网。那么看到好的东西要想凭记忆记住真的太难了。我自身也不拥有那种过目不忘的能力。所以当看到好的内容，能够快速的收藏起来，再到用的时候，能够快速的找到，则会提高我们的学习和工作的效率。这里有两个软件推荐，一个是国内的勺子收藏夹，另一个是国外的Pocket。这两个我更喜欢勺子收藏夹。因为这个界面我更喜欢。不过这个无伤大雅，但是一定要有一个。学会把有用的收藏起来。

而且对于自己觉得非常有用的内容，为了防止内容在互联网上删除，记得备份到本地一份。奥对了，忘了讲，这里以及上面的周总结和学习缓冲区，我都是使用的印象笔记记录，其实这个也没有什么关系，这种软件也很多，随便找一个来用就行了，大同小异，但一定要有一个。这里选中有用的内容，包括图片，直接复制粘贴到印象笔记中就可以了。

做笔记

笔记在我看来有两种。一种是读书读文章的笔记。一种是零散的笔记。

有的时候我们读书或者读文章，这种情况内容通常会比较长，那么当我们遇到问题再想找这里面的有用的东西，你要再读一遍吗？当然，如果你没有做笔记，只能凭借自己个印象，去全文中查找，如果是纸质书你更不可能Ctrl+F(网页中查找的快捷键，通常在网页中查找关键字时使用)。但是，如果你做了笔记，那么速度将会提高数倍。而且你记过的笔记也会印象更加深刻。所以我不仅对于书会做笔记，对于网上比较长的文章也会去做笔记。

另外一种是比较零散的笔记。这种笔记往往是一些小细节，比如：LinkedList别在循环中一直get(i)呀，if语句先把两个分支都写出来再简化呀，不要轻易的把一个类内部的List的引用让外部get到，如果有必要，使用Collections的unmodifiableList方法为其提供一个不可变的List呀……这些小的细节，遇到一个添加一个，时不时拿出来看一遍，加深印象。

结束语

现在想想自己在学校混的日子真的是太浪费了。希望刚上学的你能珍惜学生时代的时光。也希望像我一样脱离了学生时代的你能以更好的状态去学习。

你也根据自身的特点去设计一套学习方法吧。再次奉劝，长的帅的和有女朋友的手下留情，不然真的要咬人了!

Android中你还在使用HashMap<Integer,Object>吗？

众所周知，当我们要维护一个整型到对象的映射关系的时候，想定义一个Map<int,Object>会报错，我们必须使用Map<Integer,Object>。

明明只需要使用一个整型数据，却要使用一个类。这并不是杀鸡用牛刀，而是一种浪费。

是不是很押韵。

SparseArray的简介

SparseArray，是android.util包下的一个类，介绍是这样的：

SparseArrays map integers to Objects. Unlike a normal array of Objects, there can be gaps in the indices. It is intended to be more memory efficient than using a HashMap to map Integers to Objects, both because it avoids auto-boxing keys and its data structure doesn’t rely on an extra entry object for each mapping.

意思就是(翻译的不好，勿喷)：

SparseArrays是一个整型到对象的映射，它不像一个正常的对象数组，在索引处可以是空的。他是为了比HashMap提供更高的内存性能而设计的。原因有两点：

  1. 它避免了对key自动装箱操作；
  2. 每个映射关系也不是依赖额外的对象。
